## 1. Обзор проекта

**McGill** — мобильная прогрессивная веб-платформа (PWA), объединяющая фитнес-инструменты, языковое обучение и коллекцию мини-игр. Центральный хаб (`index.html`) управляет навигацией, авторизацией, статистикой и данными. Все приложения загружаются внутри `<iframe>` и взаимодействуют с платформой через единый `PlatformAPI`.

Всё работает **без сервера** — чистый HTML/CSS/JS, данные хранятся в Google Sheets, хостинг на GitHub Pages.

| Модуль | Кол-во | Описание |
|---|---|---|
| 🏋️ Фитнес | 2 | Таймер тренировок + конструктор программ |
| 📖 Обучение | 1 | Словарь с карточками, тестами, рейтингом |
| 🎮 Игры | 17 | Аркады, головоломки, логика |

### Навигация и вкладки

Платформа имеет 5 основных вкладок. На десктопе отображаются в боковой панели, на мобильных — в нижней навигации.

| Вкладка | Иконка | Описание |
|---|---|---|
| **Приложения** | 📱 | Сетка всех приложений с поиском. Показывает время последнего запуска. |
| **Статистика** | 📊 | Какой пользователь, какое приложение запускал, сколько раз, сколько времени провёл. Фильтры по приложению и пользователю. |
| **Рейтинг** | 🏆 | Рейтинг пользователей по суммарному времени в приложениях. Раскрываемая детализация по приложениям. |
| **Справочники** | 📚 | Полный CRUD-доступ ко всем листам Google Sheets. Сгруппированы по префиксу (sys_, dict_). Создание/удаление листов, строк, редактирование ячеек. |
| **Настройки** | ⚙️ | 4 подвкладки: Общие (аккаунт, БД, юзеры), Кэш (CRUD по localStorage), Обновление (SW, кэши), Разработчику (промт-инструкция). |

### PlatformAPI

Приложения обращаются к платформе через `window.parent.PlatformAPI`:

```javascript
const Platform = window.parent?.PlatformAPI;

// Пользователь
Platform.user.getProfile()   // → { login, role, total_xp }
Platform.user.isAdmin()      // → true/false
Platform.user.getRole()      // → "admin" | "user"

// База данных
Platform.db.read(range)
Platform.db.append(range, values)
Platform.db.update(range, values)
Platform.db.batchValues(data)
Platform.db.batchStructural(requests)
Platform.db.ensureTables([{ name, headers }])
Platform.db.getSpreadsheetId()

// Очки
Platform.gameplay.submitScore(appId, rawScore, xpEarned)

// Логирование (сессии трекаются автоматически платформой)
Platform.system.logSession(appId, durationSec)
Platform.system.log(appId, eventType, details)

// UI
Platform.ui.showToast(msg, type)   // type: 'ok' | 'err' | ''
Platform.ui.showLoader(on)
Platform.ui.openSettings()
```

### База данных (Google Sheets)

Платформа хранит данные в Google Sheets через Service Account JWT (RS256).

**Системные таблицы (создаются платформой):**

| Лист | Столбцы | Назначение |
|---|---|---|
| `sys_Users` | ID, Email, Password, Role, TotalXP, CreatedAt, Status | Аккаунты пользователей |
| `sys_Logs` | Timestamp, Email, AppID, EventType, Details | Журнал событий и сессий |
| `sys_Scores` | Timestamp, Email, AppID, RawScore, XP | Очки и достижения |

**Таблицы Словаря (создаются приложением):**

| Лист | Столбцы | Назначение |
|---|---|---|
| `dict_languages` | code, name | Языки |
| `dict_categories` | id, name, icon | Категории слов |
| `dict_words` | category, lang_from, word_from, lang_to, word_to, transcription | Словарный запас |
| `dict_ratings` | login, words_learned, quizzes_done, quiz_score, last_active | Рейтинг |

> Любое приложение может создавать свои листы через `Platform.db.ensureTables()`. Рекомендуемый формат: `appname_tablename` (только `[a-zA-Z0-9_]`).

### Система пользователей

- Первый зарегистрированный пользователь автоматически получает роль `admin`
- Роли: `admin` (полный доступ) и `user` (базовый доступ)
- Администратор управляет ролями и блокировкой через **Настройки → Общие → Управление пользователями**
- Платформа — единственный источник ролей; приложения используют `Platform.user.isAdmin()`

## 3. Первоначальная настройка

### 1. Создайте Google Таблицу

Откройте [Google Sheets](https://sheets.google.com) и создайте новую таблицу. ID таблицы — часть URL:
```
https://docs.google.com/spreadsheets/d/ВОТ_ЭТОТ_ID/edit
```

### 2. Создайте сервисный аккаунт

1. Откройте [Google Cloud Console](https://console.cloud.google.com)
2. Создайте проект (или выберите существующий)
3. Включите **Google Sheets API**: *APIs & Services → Library → Google Sheets API*
4. Создайте сервисный аккаунт: *IAM & Admin → Service Accounts → Create Service Account*
5. Создайте JSON-ключ: *Service Account → Keys → Add Key → JSON*
6. Скачайте файл — это и есть «JSON сервисного аккаунта»

### 3. Откройте доступ к таблице

В таблице нажмите **«Поделиться»** и добавьте email сервисного аккаунта (`...@...iam.gserviceaccount.com`) с ролью **«Редактор»**.

### 4. Настройте платформу

1. Откройте приложение (первый запуск покажет экран настройки)
2. Введите **Spreadsheet ID** и **JSON сервисного аккаунта**
3. Нажмите **«Сохранить и проверить»** — платформа создаст системные листы
4. Зарегистрируйте первый аккаунт — он станет администратором

### 📖 Словарь

| | |
|---|---|
| **Файл** | `apps/dictionary.html` |
| **Категория** | ⭐ Приложения (встроенное) |
| **Описание** | Изучение иностранных слов через карточки и тесты |
| **Таблицы БД** | `dict_languages`, `dict_categories`, `dict_words`, `dict_ratings` |

**Вкладки приложения:**
- **📚 Словарь** — все слова по категориям, выбор языковой пары, поиск, прогресс (⚫ не изучено → 🟠 учу → 🟡 знаю → 🟢 выучено)
- **🎴 Карточки** — флеш-карточки с переворотом и свайпами (вправо — знаю, влево — учить), обратный режим
- **❓ Тест** — квиз из 10 вопросов с 4 вариантами, автообновление рейтинга
- **🏆 Рейтинг** — лидерборд по выученным словам и % правильных ответов
- **⚙️ Настройки** — CRUD языков/категорий, импорт/экспорт JSON, массовый текстовый импорт, управление пользователями (admin)

**Импорт/Экспорт:**
- JSON-экспорт по языковой паре с возможностью скачать файл или скопировать
- Умный JSON-импорт: новые → добавляются, изменённые → обновляются, дубликаты → пропускаются
- Массовый текстовый импорт в формате `слово|перевод|транскрипция`

**Роли:**
- `user` — изучение, карточки, тесты, рейтинг
- `admin` — + CRUD языков/категорий/слов, управление пользователями

## 6. PWA и офлайн-режим

- **Установка:** на Android/iOS нажмите «Добавить на главный экран» в браузере
- **Версия кэша:** `app-v4`

**Стратегия кэширования (Service Worker):**

| Тип | Стратегия | Файлы |
|---|---|---|
| Оболочка | Cache First | `index.html`, `manifest.json`, иконки |
| Приложения | Network First | Все `apps/*.html`, `workouts/*.json` |
| Внешние API | Bypass (не кэшируются) | Google Sheets API, OAuth, CDN |

**Команды SW (через postMessage):**
- `CLEAR_CACHE` — очистить все кэши
- `GET_CACHE_INFO` — информация о кэше
- `SKIP_WAITING` — активировать обновление

## 8. Разработка нового приложения

### Шаблон

```html
<!DOCTYPE html>
<html lang="ru">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width,initial-scale=1">
  <title>My App</title>
</head>
<body>
  <div id="app">Загрузка...</div>
  <script>
    const Platform = window.parent?.PlatformAPI || null;

    async function init() {
      if (!Platform) {
        document.getElementById('app').textContent = 'Откройте через платформу McGill';
        return;
      }
      const profile = Platform.user.getProfile();
      if (!profile) {
        document.getElementById('app').textContent = 'Войдите в платформу';
        return;
      }

      // Создание таблиц (один раз)
      await Platform.db.ensureTables([
        { name: 'myapp_data', headers: ['Col1', 'Col2'] }
      ]);

      document.getElementById('app').textContent = 'Привет, ' + profile.login + '!';
    }

    init();
  </script>
</body>
</html>
```

### Регистрация приложения

Добавьте запись в `apps/index.json`:

```json
{
  "id": "myapp",
  "name": "Моё Приложение",
  "icon": "🎮",
  "desc": "Описание приложения.",
  "file": "myapp.html"
}
```

### Правила именования таблиц

- Формат: `prefix_tablename` (например `myapp_scores`)
- Разрешены только: `[a-zA-Z0-9_]`
- Системные таблицы: `sys_Users`, `sys_Logs`, `sys_Scores`
- Таблицы словаря: `dict_*`
